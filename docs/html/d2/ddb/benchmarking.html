LKM for exchanging messages between threads
Benchmarking

The user-level tool can be also used for benchmarking purpose, in fact it is possible to pass an INI files that contains commands to the tool. The list of the available commands and sections can be found below.

  • Section: group
    1. install=group_name: install a new group with "group_name" as descriptor
    2. loadDescriptor=group_name: load an existing group with "group_name" descriptor
    3. loadID=ID: load an existing group with id equal to ID
  • Section: message
    1. write=message: write the message "message" into the previously loaded group
    2. read=size_to_read: read "size_to_read" bytes from a previously loaded group
    3. delay=value: set the delay of the currently loaded group to "value"
    4. revoke_delay=1: revoke all the delayed message of the current group
    5. flush=1: insert all the delayed message into the FIFO queue
  • Section: synch
    1. sleep=1: put the current process on sleep
    2. awake=1: awake all the process of a group which was previously put on sleep
  • Section: config
    1. max_message_size=value: set the 'max_message_size' param to "value"
    2. max_storage_size=value: set the 'max_storage_size' param to "value"
    3. garbage_collector_ratio=value: set the 'garbage_collector_ratio' param to "value"
    4. include_structure_size=value: set the 'include_structure_size' param to "value" (0 or 1)
    5. read_test=1: read sequentially all the group's parameters from sysfs (useful for performance checks)
  • Section: security
    1. strict_mode=value: set the strict mode flag to "value" (0 or 1)
    2. change_owner=PID: change the current group's owner to "PID"
  • Section: testing
    1. fork=value: fork the current process in order to generate up to "value" childs. Very useful to test concurrency access.

See the "config.ini" file inside the tool directory for an example.
